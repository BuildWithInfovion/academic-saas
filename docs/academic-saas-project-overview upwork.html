What We Built

One Platform. Every School Operation.

Most schools run on spreadsheets, disconnected tools, and manual paperwork. This platform replaces all of that — a single system that handles every operational workflow, built specifically for how Indian schools work.

🎓

Student Management

Admissions, profiles, status tracking (active / transferred / alumni), demographic records, and incoming TC from previous school.

🏫

Academic Structure

Hierarchical class setup — boards, streams, grades, divisions. Academic years with current-year tracking. Subject assignment per class.

📋

Attendance

Session-based daily attendance with present / absent / late / half-day statuses. Per-class sessions, bulk marking, and analytics.

📝

Exam & Results

Multi-subject exams with per-subject passing marks. Result entry, pass/fail analytics, and promotion eligibility calculation.

💰

Fee Management

Fee structures per class and academic year. Payment recording, dues tracking, and outstanding balance calculation per student.

📄

Transfer Certificate (TC)

Government-approved 21-field TC workflow — request, approval, issue. Atomic TC number generation. Auto-populated from records.

👤

Staff & Role Management

7 role types with granular permissions. Portal access for teachers, parents, receptionists, accountants, and management.

🔒

Multi-School Isolation

Each school on the platform is completely isolated. No data crossover. Institution-level tenancy enforced at every layer.

Key Module Deep Dive

Transfer Certificate Workflow

TC issuance is a government-regulated process in India. The platform handles the full lifecycle — from request to printed document — with proper approval controls and data immutability.

1

Student Request

Operator opens student profile and initiates TC request

›
2

Operator Form

Fill all 21 TC fields — personal, academic, fee, conduct

›
3

Principal Approval

Approve or reject with remark. Rejected can be resubmitted

›
4

Issue & Print

TC number generated. Student marked transferred. Document ready to print

Government-Approved 21-Field Format

  • Name, Mother's & Father's Name
  • Nationality, Religion, Caste/Category
  • Gender, Blood Group
  • Date of first admission with class
  • Date of Birth (figures & words)
  • Class last studied
  • Annual exam taken with result
  • Subjects studied
  • Promotion eligibility
  • Fees paid up to month
  • Working days & days present
  • General conduct
  • Application & issue dates
  • Reason for leaving
Data Integrity
  • All student data captured as immutable snapshot at request time
  • TC number generated atomically inside DB transaction
  • Duplicate TC blocked — one active TC per student at a time
  • Re-request blocked once student is marked as transferred
  • School letterhead auto-populated from institution record
  • Fee dues automatically flagged during TC request
  • Attendance auto-computed from session records
  • Exam pass/fail auto-detected from result records
Access Control

Role-Based Permissions

Every user has a role that controls exactly what they can see and do. Roles are enforced at both the API layer and the UI layer.

Role Access Level Key Capabilities
Director (Super Admin) Full access All modules, all schools, user management, system config
Principal School-wide Approve TCs, view all reports, manage staff, academic setup
Teacher Class-level Mark attendance, enter exam results, view assigned class
Receptionist Front desk Student admissions, TC requests, fee collection, enquiries
Accountant Finance Fee structures, payment records, dues reports
Non-Teaching Staff Limited View-only access to assigned functions
Parent Own child only View child's attendance, results, fee status via parent portal
Multi-Tenant Security

Complete School Data Isolation

  • Every DB row tagged with institutionId
  • Tenant middleware validates X-Institution-ID header
  • AuthGuard + RolesGuard on every API route
  • No cross-school data leakage possible
  • School A cannot read, write, or access School B's data
  • TC numbers unique per school, not globally
